# 00. 과제 개요 및 요약

> 출처: 과제 명세 + `README.md`

## 평가의 핵심

> **"코드를 고치는 능력"이 아니라 "왜 그렇게 결정했는가"를 본다.**

- 정답이 하나가 아니다 → **판단과 트레이드오프 분석**이 핵심 평가 대상
- 수정마다 **의사결정 근거**를 남겨야 함
  - GitHub 제출: **커밋 메시지 + PR 코멘트**
  - 공통: **`REPORT.md`** 에 근거 정리
- **AI 도구 사용 시 README에 사용 범위·참고 내용 기재 필수**

## 정성 평가 체크포인트 (README 발췌)

## 과제 구성

### Part A — Flutter 코드 디버깅
정상 동작해야 하는 테스트 앱에 **의도적으로 심어둔 버그**들을 찾아 수정.
`REPORT.md` 표가 5행 → 의도된 버그가 **약 5개**로 추정.

앱이 만족해야 하는 정상 동작 (과제 화면 설명):
1. **닉네임 변경** — 우측 상단 사람 아이콘 → 닉네임 변경 시 홈 화면에 즉시 반영
2. **즐겨찾기** — 상담사 목록 하트 아이콘으로 등록/삭제
3. **채팅방** — 입장 시 기본 메시지 표시, **10초마다 상담사 메시지 수신**, **직접 입력한 메시지 전송 가능**

→ 상세 분석: [`01-part-a-analysis.md`](01-part-a-analysis.md)

### Part B — MySQL 쿼리 개선
관리자 페이지용 쿼리 2개의 성능 문제·설계 결함 진단 → 개선 쿼리 + `CREATE INDEX` 작성.
- **B-2-1**: 최근 30일 일별 상담 완료 통계 (현재 응답 5초+ 이슈)
- **B-2-2**: 온라인 상담사별 누적 완료 상담 횟수 N+1 쿼리 → 단일 쿼리화

스키마: `users`(~50만), `counselors`(~5천), `chat_rooms`(~200만). **현재 인덱스는 PK만 존재.**

→ 상세 분석: [`02-part-b-analysis.md`](02-part-b-analysis.md)
